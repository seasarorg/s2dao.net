S2Dao.NET Setup

To use S2Dao.NET, reference to the following files (assemblies) should be set. Make Seasar.Dao.Dll in build folder and ja-JP folder to be the same.

  • s2dao.net/build/Seasar.Dao.dll

S2Dao.NET requires S2Container.NET(Seasar.dll) and log4net. Refer to the following pages for setup instructions on these software.

  • S2Container.NET Setup
  • log4net Setup

Specify S2Dao.NET assembly(Seasar.Dao) in application configuration file (app.config). 

<configuration>
   <configSections>
      <section name="seasar"
         type="Seasar.Framework.Xml.S2SectionHandler, Seasar" />
   </configSections>

   <seasar>
      <!-- set in  SingletonS2ContainerFactory#Init the path of dicon file
below as the initial value of the root dicon file. -->
      <configPath>App.dicon</configPath>
      
      <!-- During creation of S2Container, read in assembly below into AppDomain
(S2ContainerFactory#Create) -->
      <assemblys>
         <assembly>assembly name 1</assembly>
         <assembly>assembly name 2</assembly>
         <assembly>Seasar.Dao</assembly>
      </assemblys>
   </seasar>

</configuration>

Refer to the S2Container.NET reference - application configuration file for further details.
